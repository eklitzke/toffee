toffee: technical details

toffee runs off of a custom dns server built using the excellent libevent non-blocking i/o library. libevent version 2 contains an api for building dns servers, and that's what toffee uses.

why a custom dns server?

the simple answer why we use a custom dns server is that it's the simplest and most efficient way to provide an extremely dynamic custom dns service like this. most dns servers are designed for organizations with a relatively static infrastructure. programatically updating dns configuration for dns servers like bind and djbdns is difficult and awkward. it frequently involves writing complex scripts that generate dns databases and rsync these databases around.

bugs

we take the speed, reliability, and correctness of toffee very seriously. we commit to resolving any bugs you find in toffee immediately; at the time of this writing, there are no known bugs in toffee. if you notice dns features which are not fully supported by toffee, or experience any other technical problems, please let us know by contacting us at support@toffeedns.com.
